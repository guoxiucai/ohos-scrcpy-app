# Feature Specification: ohscrcpy CLI 远程控制工具

**Feature Branch**: `001-ohscrcpy-cli`

**Created**: 2026-05-21

**Status**: Draft

**Input**: 在本项目创建一个cli目录，在其中实现一个可以至少运行在mac的cli ohscrcpy，基于本项目原理实现OpenHarmony设备远程控制，参数参考scrcpy，服务端自动安装，仅需远程控制功能，右键=返回键，设备选择通过参数实现。

---

## Clarifications

### Session 2026-05-21

- Q: `ohscrcpy` 是否需要支持 `--verbose` / `-v` 详细日志参数，在 stderr 输出连接状态、帧率统计、控制事件等调试信息？ → A: 是，支持 `--verbose` / `-v`，输出连接流程、视频帧统计、控制事件等调试日志到 stderr。
- Q: 针对同一设备，当已有 `ohscrcpy` 实例正在运行时再次启动应如何处理？ → A: 检测到端口冲突或已有同设备实例时，打印明确错误信息并退出，不强行抢占。

## 用户场景与测试 *(mandatory)*

### 用户故事 1 - 连接设备并投屏远程控制 (Priority: P1)

开发者或测试人员将 OpenHarmony 设备通过 USB 连接到 Mac，在终端执行 `ohscrcpy` 命令后，自动完成服务端检测与安装，随后弹出一个显示设备屏幕的窗口，并可通过鼠标对设备进行操作。

**Why this priority**: 这是工具的核心价值，没有投屏和交互功能，工具无法使用。

**Independent Test**: 连接设备后执行 `ohscrcpy`，可独立验证窗口弹出、画面显示、鼠标点击响应。

**Acceptance Scenarios**:

1. **Given** 设备通过 USB 连接且服务端已安装，**When** 执行 `ohscrcpy`，**Then** 弹出投屏窗口，显示设备实时画面，鼠标点击有效传递触控事件。
2. **Given** 设备通过 USB 连接但服务端未安装，**When** 执行 `ohscrcpy`，**Then** 自动安装服务端 HAP 后再弹出投屏窗口，全程无需手动操作。
3. **Given** 投屏窗口已打开，**When** 用户在窗口内点击鼠标右键，**Then** 向设备发送返回键事件，设备界面退回上一页。

---

### 用户故事 2 - 通过参数选择指定设备 (Priority: P2)

当开发者同时连接多台 OpenHarmony 设备时，通过 `-s <serial>` 参数指定要控制的目标设备。

**Why this priority**: 多设备场景下精确选择目标设备是正确工作的前提，单设备时可自动选择。

**Independent Test**: 同时连接两台设备，使用 `-s <serial>` 分别控制各自设备，验证投屏窗口对应正确的设备。

**Acceptance Scenarios**:

1. **Given** 连接了多台设备，**When** 执行 `ohscrcpy -s <serial>`，**Then** 仅连接并控制指定序列号的设备，其他设备不受影响。
2. **Given** 连接了一台设备，**When** 执行 `ohscrcpy`（不带 `-s`），**Then** 自动连接该唯一设备。
3. **Given** 连接了多台设备但未指定 `-s`，**When** 执行 `ohscrcpy`，**Then** 打印错误提示并列出可用设备序列号，不启动投屏。

---

### 用户故事 3 - 调整画面分辨率与码率 (Priority: P3)

开发者通过参数调整投屏画面的最大分辨率和视频码率，在画质与性能之间取得平衡。

**Why this priority**: 基础功能能用后，画面参数调整是提升体验的重要选项。

**Independent Test**: 使用 `--max-size 1024` 和 `--bit-rate 4M` 参数启动，验证窗口尺寸和视频质量符合预期。

**Acceptance Scenarios**:

1. **Given** 执行 `ohscrcpy --max-size 1024`，**When** 投屏建立，**Then** 投屏画面的长边不超过 1024 像素。
2. **Given** 执行 `ohscrcpy --bit-rate 2M`，**When** 投屏运行，**Then** 视频流编码码率接近 2 Mbps。
3. **Given** 未提供任何参数，**When** 启动投屏，**Then** 使用默认分辨率（不裁剪）和默认码率（8 Mbps）。

---

### 边界与异常情况

- 没有检测到任何已连接的 OpenHarmony 设备时，打印明确错误信息并退出，不崩溃。
- 服务端 HAP 自动安装失败时（如签名不匹配），打印错误原因并提示手动安装步骤，退出。
- 端口转发（hdc fport）失败时，提示端口冲突或权限问题，并给出排查建议。
- hdc 工具不在系统 PATH 且无内置版本时，打印清晰的依赖缺失错误，并提供安装指引。
- 投屏过程中设备断连，窗口显示连接中断提示，进程正常退出（不崩溃）。
- 多台设备在未指定 `-s` 时给出可用设备列表，引导用户重新执行命令。
- 针对同一设备已有实例在运行时再次启动，打印端口冲突错误（含已占用端口号）并退出，不影响已有实例。

## 需求 *(mandatory)*

### 功能性需求

- **FR-001**: 工具 MUST 提供命令行界面，可执行文件名为 `ohscrcpy`，至少支持 macOS（Apple Silicon 和 Intel）。
- **FR-002**: 工具 MUST 在启动时检测目标设备上是否已安装服务端（`com.ohos.scrcpy.server`），若未安装或版本不符则自动安装内置的 HAP 包，无需用户手动操作。
- **FR-003**: 工具 MUST 通过 `hdc fport` 建立 PC 端口与设备 53535 端口之间的 TCP 转发通道，并在退出时自动清理转发规则。
- **FR-004**: 工具 MUST 启动一个原生桌面窗口，实时渲染设备的 H.264 视频流，帧率不低于设备推流帧率。
- **FR-005**: 工具 MUST 将窗口内的鼠标左键按下/移动/抬起事件转换为设备触控事件并通过协议发送到设备。
- **FR-006**: 工具 MUST 将窗口内的鼠标右键单击事件映射为设备返回键（BACK）控制指令。
- **FR-007**: 工具 MUST 支持 `-s <serial>` / `--serial <serial>` 参数，用于在多设备场景下指定目标设备。
- **FR-008**: 工具 MUST 支持 `--max-size <px>` 参数限制投屏画面的最大边长（像素），默认不限制（使用设备原始分辨率）。
- **FR-009**: 工具 MUST 支持 `--bit-rate <value>` 参数设定视频流的目标码率，支持 `K`/`M` 后缀（如 `2M`），默认 8M。
- **FR-010**: 工具 MUST 支持 `--fps <value>` 参数设置目标帧率，默认 60。
- **FR-011**: 工具 MUST 在未指定 `-s` 且存在多台已连接设备时打印设备列表并以非零状态码退出。
- **FR-012**: 工具 MUST 支持 `--version` 参数打印版本号后退出。
- **FR-013**: 工具 MUST 支持 `--help` / `-h` 参数打印用法说明后退出。
- **FR-014**: 服务端 HAP 包 MUST 内置在工具发布包内，不依赖外部网络下载。
- **FR-015**: 工具不需要实现设备 Shell 功能、物理按键模拟（除返回键外）、设备状态面板等辅助功能。
- **FR-016**: 工具 MUST 支持 `--verbose` / `-v` 参数，启用后将连接流程、帧率统计、控制事件等调试信息输出到 stderr；默认不输出调试信息。
- **FR-017**: 工具 MUST 在检测到端口转发冲突或针对同一设备已有运行实例时，打印明确错误信息（含已占用端口号）并以非零状态码退出，不强行抢占已有连接。

### 关键实体

- **Device（设备）**: 通过 USB 连接的 OpenHarmony 设备，由序列号唯一标识；关键属性：序列号、屏幕分辨率、服务端安装状态。
- **ScrcpyServer（服务端）**: 运行在设备上的 ServiceExtensionAbility（bundleName=`com.ohos.scrcpy.server`），负责采集屏幕、编码 H.264 并监听 TCP 53535。
- **VideoStream（视频流）**: H.264 Annex-B 格式的实时码流，由服务端推送到客户端，通过协议帧承载。
- **ControlEvent（控制事件）**: 由客户端发送到服务端的操作指令，包括触控事件（type 0x10, subType 0x01–0x03）和返回键（type 0x10, subType 0x10）。
- **PortForward（端口转发）**: 由 `hdc fport` 建立的 PC 端口→设备 53535 的映射规则，生命周期与 ohscrcpy 进程绑定。

## 成功标准 *(mandatory)*

### 可度量的结果

- **SC-001**: 从执行 `ohscrcpy` 到投屏画面首帧显示，在服务端已安装情况下耗时不超过 5 秒。
- **SC-002**: 服务端未安装时，自动安装完成并显示首帧的端到端时间不超过 30 秒。
- **SC-003**: 投屏画面帧率达到参数设定值的 90% 以上（默认 60 fps 时实测不低于 54 fps）。
- **SC-004**: 鼠标点击到设备响应的端到端延迟不超过 200 ms（局域网/USB 正常状态下）。
- **SC-005**: 工具在运行时 CPU 占用不超过单核的 80%，内存占用不超过 200 MB。
- **SC-006**: 在 10 次重复连接/断连测试中，工具每次均能正常退出且不留残留进程或端口转发规则。
- **SC-007**: `--help` 输出涵盖所有已实现参数，首次使用者无需查阅额外文档即可完成基本操作。
- **SC-008**: 使用 `--verbose` 模式时，stderr 输出的连接耗时、首帧时间、实时帧率信息足以在不借助其他工具的情况下诊断常见连接问题。

## 假设

- 目标平台首先为 macOS（包括 Apple Silicon M 系列和 Intel x86_64）；Linux/Windows 为后续支持目标。
- 用户系统已安装 `hdc` CLI 工具（华为 DevEco Studio 随附），或项目内置 hdc 二进制文件可作为回退。
- 设备通过 USB 连接，hdc 可正常识别设备；不支持纯网络 hdc 场景（TCP 直连 LAN 设备）作为 v1 目标。
- 服务端 HAP 使用 debug 签名，目标设备处于开发者模式（允许安装 debug 应用）。
- 工具以单进程形式运行，不提供后台守护服务或 SDK 接口。
- 投屏窗口为独立原生窗口（macOS 上使用系统原生渲染），不内嵌于其他应用。
- 不需要处理设备屏幕旋转的动态切换（v1 按设备当前方向渲染，不自动旋转）。
